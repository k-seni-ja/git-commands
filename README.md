# **Основные команды Git:**

## 1. Настройка Git на новом устройстве
|Команда git|Примечание|
|-----------|----------|
|git config --list|проверка текущих настроек git|
|git config --global user.name "Ваше Имя"|-|
|git config --global user.email "ваш@email.com"|-|

## 2. Создание и клонирование репозитория
|Команда git|Примечание|
|-----------|----------|
|git init|Инициализировать локальный репозиторий|
|git clone <url>               |Скачать копию удаленного репозитория (в папку с названием репозитория)|
|git clone <url> <папка>       |Скачать копию удаленного репозитория (в папку с другим названием)|
|git clone <SSH-адрес>         |Скачать копию удаленного репозитория (безопасное зашифрованное соединение)|

## 3. Основной рабочий процесс в репозитории
|Команда git|Примечание|
|-----------|----------|
|git status                   |Показать состояние репозитория на данный момент|
|git add <файл>                |Добавить конкретный измененный файл для сохранения|
|git add .                     |Добавить все изменения в текущей папке и подпапках для сохранения|
|git add -А                    |Добавить все изменения во всем репозитории для сохранения|
|git add -p                    |Интерактивно выбрать, какие части изменений в файле добавить для сохранения|
|git commit -m "сообщение"     |Закоммитить изменения с коротким комметарием|
|git commit                    |Открыть текстовый редактор для ввода подробного описания коммита|
|git commit --amend            |Исправить последний коммит. Только для коммитов, которые еще не были отправлены на удаленный сервер (git push)|

## 4. Ветки (branches)
|Команда git|Примечание|
|-----------|----------|
|git branch                    |Показать все ветки проекта, снежинкой выделена текущая ветка|
|git branch -a                 |Показать все ветки проекта, локальные и удаленные|
|git branch <имя>              |Создать новую ветку|
|git branch -d <имя>           |Удалить ветку безопасно (не даст удалить, если изменения не слиты)|
|git branch -D <имя>           |Удалить ветку принудительно|
|git checkout <имя>            |Переключиться на ветку|
|git checkout -b <имя>         |Создать ветку и сразу переключиться на нее|
|git merge <присоединяемая ветка>  |Слить ветку в текущую|

## 5. Удаленные репозитории (remote)
|Команда git|Примечание|
|-----------|----------|
|git remote -v                 |Показать удаленные репозитории, связанные с локальным|
|git remote add origin <url>   |Добавить удаленный репозиторий|
|git push origin <ветка>       |Отправить изменения на удаленный репозиторий|
|git push                      |Отправить изменения на удаленный репозиторий, когда upstream уже настроен для текущей ветки|
|git pull origin <ветка>       |Загрузить изменения из удаленного репозитория со слиянием|
|git pull                      |Загрузить изменения из удаленного репозитория со слиянием, когда upstream уже настроен для текущей ветки|
|git pull -rebase              |Загрузить изменения из удаленного репозитория, переписать историю коммитов локальной ветки|
|git fetch origin              |Загрузить изменения из удаленного репозитория без слияния|

## 6. Просмотр истории
|Команда git|Примечание|
|-----------|----------|
|git log                       |Показать историю коммитов|
|git log --oneline             |Краткая история коммитов|
|git log --graph               |Графическое представление истории коммитов|
|git log --graph --oneline --decorate |Графическое представление истории коммитов|
|git log <файл>                |Показать историю изменений конкретного файла|
|git reflog                    |Показывает все ваши действия в репозитории|
|git show <хеш>                |Показать изменения коммита|
|git diff                      |Показать незакоммиченные изменения в файлах и папках|
|git diff <ветка> <ветка>      |Показать разницу двух веток|

## 7. Отмена изменений
|Команда git|Примечание|
|-----------|----------|
|git restore <файл>            |Отменить изменения в файле|
|git restore --staged <файл>   |Убрать файл из add|
|git reset --soft HEAD-1       |Отменить последний коммит, сохранив изменения (- = ~)|
|git reset --hard HEAD-1        |Полностью удалить последний коммит с внесенными  изменениями|
|git reset --hard [код из reflog]  |Вернуться к любому состоянию репозитория|
|git revert <хеш>              |Создать коммит, отменяющий изменения|

## 8. Очистка
|Команда git|Примечание|
|-----------|----------|
|git clean -n                  |Показать файлы для удаления|
|git clean -f                  |Удалить неотслеживаемые файлы|
